Estoy desarrollando una herramienta que, a partir de una descripción en lenguaje natural 
sobre una aplicación que quiero crear y otras más especificaciones ("arquitectura": 
["Frontend", "Backend", ...], "framework": ["Django", "Flask", ...]), genere automáticamente 
un diagrama UML de clases en formato Mermaid, junto con una explicación textual del diagrama. 
El objetivo es que, escribiendo una descripción de la aplicación (por ejemplo, mencionando 
las entidades, atributos, relaciones, y preferencias técnicas como frameworks o lenguajes), 
el sistema devuelva un diagrama visual y una descripción clara que ayude a entender la 
estructura de la aplicación. Además con la intención de que con el tiempo se pueda ampliar y 
también sea capaz de generar la estructura de ficheros de la aplicación.

Inicialmente, la idea era usar un modelo de lenguaje grande (LLM) como GPT-4 para interpretar 
el texto y generar directamente el diagrama y la explicación. Sin embargo, debido a costos 
asociados y para tener mayor control y precisión, decidí optar por un enfoque modular con 
modelos más pequeños y gratuitos, posiblemente fine-tunados para tareas específicas.

El pipeline ideal consta de varios pasos:

1. **Extracción de entidades y atributos:** identificar las clases (entidades) mencionadas, 
sus atributos y tipos desde la descripción en lenguaje natural.
    
2. **Extracción de relaciones:** determinar las relaciones entre esas entidades (por ejemplo, 
asociaciones, herencias, cardinalidades).
    
3. **Generación de estructura intermedia:** representar la información extraída en un formato 
estructurado estándar (JSON) que describa clases, atributos y relaciones.
    
4. **Generación de código Mermaid:** transformar el JSON en código válido Mermaid para la 
visualización del diagrama.
    
5. **Generación de descripción textual:** crear un texto explicativo que describa el diagrama 
generado, facilitando la comprensión para el usuario.
    

Para lograr esto, planeo crear y entrenar varios modelos especializados, principalmente para 
las tareas de extracción (entidades y relaciones), mientras que la generación de Mermaid puede 
hacerse con reglas o plantillas, y la explicación puede ser generada con modelos más simples 
o mediante plantillas.

Ahora necesito ayuda para generar un pequeño dataset que sirva para entrenar estos modelos de 
extracción. El dataset debe contener ejemplos de descripciones en lenguaje natural junto con 
las anotaciones correctas de las entidades, atributos y relaciones para que los modelos 
aprendan a reconocerlas. También quiero que este dataset sea lo suficientemente variado 
para cubrir distintos tipos de aplicaciones y estructuras, y que facilite el entrenamiento y 
la validación automática.
