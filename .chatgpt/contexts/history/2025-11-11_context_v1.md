2n Contexto

AI App Blueprint Visualizer es una herramienta que, a partir de una descripción en lenguaje natural de una
aplicación, genera automáticamente:

- Una estructura de carpetas coherente
- Un análisis con las tecnologías recomendadas
- Una visualización técnica de la arquitectura (diagrama)

Por ejemplo, el usuario puede escribir:

> “Quiero una app para gestionar reservas de restaurantes”

Y el sistema devolverá:

- Un análisis semántico del tipo de app
- Un diagrama arquitectónico en Mermaid.js
- Una estructura base de archivos para el backend y frontend
- Recomendaciones tecnológicas (lenguajes, frameworks, bases de datos)

Stack Tecnológico Inicial
Componente -> Tecnología
Backend FastAPI -> (Python)
Frontend -> HTML + JavaScript + Tailwind
IA -> Modelo open source (LLaMA 3, Mistral 7B) con posible fine-tuning
Visualización -> Mermaid.js o D3.js
Persistencia -> JSON o SQLite (simple y portable)

Enfoque de desarrollo

- Calidad arquitectónica como prioridad
- Iteración rápida, validando en pequeñas entregas semanales
- Documentación clara y mantenible
- Enfoque de desarrollo modular, con posibilidad de expansión futura

Flujo general de uso

1. El usuario introduce una descripción textual de su app
2. El sistema procesa esta entrada con un modelo de IA open source
3. Se genera:
   - Análisis funcional y técnico
   - Diagrama arquitectónico (Mermaid.js)
   - Sugerencias tecnológicas
   - Estructura base del proyecto
4. El frontend visualiza los resultados en una UI clara y visual

Consideraciones estratégicas

- Este proyecto forma parte de una estrategia personal para construir un ecosistema de herramientas tecnológicas propias
- Eventualmente, estos proyectos se integrarán en una corporación tecnológica personal
- El código está siendo diseñado para ser modular, extensible y mantenible

Estado actual
Semana 1 – Enfoque en:

- Definición de la arquitectura general del sistema
- Diseño de la estructura de carpetas
- Estrategia de uso de IA (solo modelos open source, sin GPT-4)
- Endpoints iniciales: analyze, generate_structure, generate_full_blueprint

Objetivo a corto plazo

- Tener una primera versión funcional capaz de:
- Recibir descripciones de apps del usuario
- Generar análisis + diagrama Mermaid.js
- Crear una estructura inicial del proyecto basada en el análisis
- Mostrar todo esto en una interfaz clara y funcional

1r Contexto

Estoy desarrollando un proyecto llamado AI App Blueprint Visualizer.

El objetivo es crear una herramienta que, a partir de una descripción en lenguaje natural de una aplicación,
genere automáticamente:

- una estructura de carpetas coherente,
- las tecnologías recomendadas,
- y una visualización del diseño técnico de la app.

El usuario podrá escribir “quiero una app para gestionar reservas de restaurantes” y el sistema analizará la
descripción, propondrá una arquitectura y mostrará el resultado en una interfaz visual.
El stack inicial será:

- Backend: FastAPI (Python)
- Frontend: HTML + JS + Tailwind
- IA: modelo open source tipo Llama 3 o Mistral 7B, con posibilidad de fine-tuning
- Visualización: Mermaid.js o D3.js
- Persistencia: JSON o SQLite

El proyecto forma parte de mi estrategia para construir una base sólida de proyectos propios que más adelante
formarían parte de una corporación tecnológica personal.
Estoy priorizando la calidad arquitectónica, la iteración rápida y la documentación clara.

Actualmente estamos en la Semana 1, centrados en definir arquitectura, estructura de ficheros y estrategia de
IA (sin usar GPT-4, solo modelos open source).
